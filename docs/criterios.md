# Criterios

Esta entrega foi organizada a partir da secao de entrega individual e das etapas do projeto da disciplina **Platforms, Microservices, DevOps and APIs**.

## O que o enunciado pede

A entrega individual deve ter um repositorio no GitHub com documentacao MkDocs dos exercicios realizados e do projeto, para envio do link via Blackboard.

Na documentacao publicada devem constar:

- nome do aluno e grupo;
- documentacao das atividades realizadas;
- codigo fonte das atividades realizadas;
- documentacao do projeto;
- codigo fonte do projeto;
- link para todos os repositorios utilizados;
- destaques para os bottlenecks implementados;
- apresentacao do projeto;
- video de apresentacao do projeto;
- uso de IA documentado quando aplicavel.

## Escopo individual

No meu caso, a contribuicao individual ficou concentrada em:

- `product-service`;
- `order-service`.

As etapas AWS, EKS, Jenkins, load testing, custos e apresentacao final sao evidencias do projeto em grupo e ficam referenciadas por este site.
